# 人话转译（技术内容可读化）

请把我上一条消息里的技术内容，转译成普通人也能顺着读懂的版本。

## 目标
- 不改变事实和结论。
- 保留关键数字、单位、条件和结论。
- 减少公式堆叠，优先解释“这一步在干嘛”。

## 输出要求
1. 先给结论摘要（3~5 行）。
2. 再按步骤解释，每步只做一件事。
3. 每个公式后立刻用一句人话解释含义。
4. 最后给“常见误区/易错点”。

## 风格约束
- 用简体中文。
- 句子短一点，避免大段术语。
- 可以保留必要公式，但不要整屏公式墙。
- 输出结构固定如下：

```markdown
**结论先看**
[简明结论]

**怎么得到的**
1. [步骤 1]
2. [步骤 2]
3. [步骤 3]

**易错点**
1. [误区 1]
2. [误区 2]
```
